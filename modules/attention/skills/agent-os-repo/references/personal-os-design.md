# 用户个人操作系统设计规范

> 来源：`Hermes_个人生活秘书_调度中枢完整说明.md`（用户发送的文档）
> 最后同步：2026-05-26

## 系统架构

```
微信 → Hermes（调度中枢）
          ├── 识别输入类型
          ├── 飞书日历（正式日程系统）
          ├── 飞书任务（正式任务系统）
          ├── 每日复盘 Skill
          ├── 飞书知识库/文件（完整事实）
          └── ima（高价值摘要/长期记忆召回层）
```

## 核心原则

1. **不重复建设已有系统** — 飞书日历/任务/复盘 Skill 都是现成的，只调用不重建
2. **飞书存完整记录** — 10个模块（每日记录/人脉/活动/项目/小红书/资料/灵感/决策/周报月报/可复用表达）
3. **ima 只存摘要** — 标题+日期+类型+摘要+关键结论+标签+飞书链接，不存全文
4. **判断标准：凡未来可能被复盘/复用/写内容/形成IP的，都同步 ima**

## ima 知识库建议结构（5个）

```
1. 我的经历库
2. 小红书内容库
3. AI Agent / 工具资料库
4. 项目与产品复盘库
5. 长期观点与决策库
```

## ima 同步格式

```
标题：

```

## 飞书 10 个模块字段（详见原文档 §9）

## 标签体系（§13）

**主题标签**：AI Agent / 个人知识库 / 小红书 / 内容增长 / Hackathon / 产品经理 / AI工具 / 创业 / 自动化 / 飞书 / ima / Hermes / 知识管理 / 个人成长 / 效率系统 / 生活秘书 / 个人操作系统

**类型标签**：每日记录 / 活动经历 / 项目复盘 / 人脉记录 / 内容选题 / 资料收藏 / 灵感洞察 / 决策记录 / 周报 / 月报

**价值标签**：可发小红书 / 可做项目 / 可复盘 / 可合作 / 可长期研究 / 待补充 / 高价值 / 低价值 / 需跟进 / 可沉淀为方法论

## 调度决策5问（§35）

每次收到微信消息，先问：
1. 这是什么类型的信息？
2. 需要调用飞书日历、飞书任务、每日复盘 Skill，还是写入飞书？
3. 是否需要创建日程/任务/提醒？
4. 是否有长期价值需要同步 ima？
5. 是否可以变成内容/复盘/项目资产/人脉资产？

## 建议 Skills（§33）

```
1. schedule_router：识别日程并调用飞书日历
2. task_router：识别任务并调用飞书任务
3. daily_capture：记录每日碎片并写入飞书
4. contact_manager：管理人脉和 follow-up
5. content_manager：管理小红书内容和选题
6. project_manager：管理项目进展和下一步
7. knowledge_manager：管理文章/工具/资料和 ima 同步
8. review_router：调用已有每日复盘 Skill，并负责周报/月报调度
9. ima_memory_sync：将高价值摘要同步到 ima
10. memory_recall：基于 ima + 飞书回答过往经历问题
```

> 注意：这些是用户建议的技能架构，目前尚未实现。涉及飞书知识库各模块的创建/维护工作流待补充。

## 内容创作偏好（§30-31）

- 风格：真实经历 / 第一人称 / 有钩子 / 有反差 / 有干货 / 有情绪
- 标题类型：反差型 / 经验型 / 失败复盘型 / 真实记录型 / 提问型 / 清单型
- 每次给 3 个标题选项

## 默认偏好（§29）

| 项目 | 默认值 |
|------|--------|
| 默认提醒时间 | 事件前30分钟 |
| 默认日报时间 | 晚上22:30 |
