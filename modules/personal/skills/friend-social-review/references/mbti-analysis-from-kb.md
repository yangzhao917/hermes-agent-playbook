# 参考：分析他人时判断自己身份的方法

## 2026-05-20 Session 教训

本次 session 中，我将朋友王娇的场景误认为是用户（杨钊）的场景，进行了错误的 MBTI 分析。

**问题根因**：分析时没有先确认"被分析的主体是谁"，直接用自己的假设套进去了。

**正确做法**：
1. 分析前先确认被分析对象（用户本人 vs 用户的朋友）
2. 如果用户是帮朋友问，直接说"我需要先确认一些背景"
3. 不要基于单次对话就给人贴性格标签

# 参考：MBTI 分析流程（基于飞书知识库）

当用户要求"通过知识库分析我是怎样的人"时：

## 步骤

1. **搜索用户 Wiki 中的关键文档**：个人介绍、播客大纲、项目分工表、会议纪要、经验记录
2. **提取行为证据**：主动牵头活动、多项目并行、资源整合风格、主动分享帮助他人
3. **结合对话上下文推断**：做事风格、沟通偏好、决策模式
4. **给结论时标注置信度**：高/中/低，说明依据

## 本次分析结果（2026-05-20）

用户判断：**ENTP**

依据：
- 黑客松从选手做到主办方（先下场再深挖）
- 多项目并行还游刃有余
- 主动社交不怯场（外校牵头，本校略缩——熟人圈评价成本意识）
- "试试看"哲学，不是想清楚再动

**注意**：MBTI 只是参考工具，单次对话样本量有限，结论应标注为"仅供参考"

# 参考：用户偏好

- **输出格式**：纯文本，不要 markdown bullet，直接给结论
- **避免**：过度分析、展开推理、背景说明，能一句话说清不用两句

## 第二次教训（2026-05-20 下午）

**问题**：用户要求"通过知识库分析我是怎样的人"时，Wiki 搜索返回空结果。

**应对策略**：
1. 先搜 Wiki 和知识库
2. 如果 Wiki 返回空，用 session_search 搜索历史 session
3. 如果都没有，告诉用户"信息不足，能否先描述一下你自己"
4. 不要基于不充足的信息硬分析

**注意**：分析的是用户本人（杨钊），不是王娇。王娇是用户的朋友。
