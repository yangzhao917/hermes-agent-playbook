# hackathon-activity-search-skill

## Overview

本 skill 用于主动检索、整理、筛选黑客松、AI 活动、开发者大赛、Agent 竞赛、创业营、Demo Day 等活动信息。

核心原则：

- 主动搜索，不默认要求用户提供材料。
- 宁可输出 `未说明 / 待确认 / 当前来源受限`，也不得编造信息。
- 没有来源支撑的信息，不得作为事实输出。
- 工具不可用时自动切换搜索通道，不假装已完成检索。
- 必须考虑模型幻觉风险，禁止根据经验补全未知字段。

---

## Trigger Conditions

当用户出现以下任一需求时，触发本 skill：

- 查找黑客松 / Hackathon。
- 查找 AI 活动 / AI Agent 活动 / 大模型比赛。
- 查找开发者大赛 / 创业营 / Demo Day。
- 查找近期可报名技术活动。
- 查找有奖金、有报销、有食宿的活动。
- 按城市、时间、赛道、主办方筛选活动。
- 查找小红书、AMD、京东、TRAE 等大厂相关活动。
- 从小红书、公众号、网页中检索活动信息。

示例：

- 最近有什么 AI 黑客松？
- 找一下杭州近期黑客松。
- 有没有有报销的比赛？
- 找 TRAE / 小红书 / AMD / 京东 相关活动。
- 适合我参加的 AI Agent 活动有哪些？

---

## Reverse Triggers

以下场景不触发本 skill：

- 只询问“黑客松是什么”。
- 只写活动复盘或宣传文案。
- 只优化小红书标题/笔记。
- 只创建用户自己的黑客松活动。
- 只分析某个已确定活动，而不是检索活动。
- 普通课程、直播、讲座，且没有比赛 / 项目 / Demo 机制。

---

## Functional Scope

本 skill 支持：

1. 主动检索活动信息。
2. 抽取活动字段。
3. 按条件过滤活动。
4. 合并重复活动。
5. 判断活动是否适合用户。
6. 在工具不可用时执行兜底检索。
7. 标注信息来源和不确定性。

本 skill 不支持：

- 不替用户报名。
- 不破解平台限制。
- 不绕过登录、权限、验证码或风控。
- 不保证活动最终有效性。
- 不编造任何活动信息。

## Search Strategy

执行步骤：

1. 识别用户需求 
  提取赛道、地点、时间、奖金、报销、主办方等条件。

2. 判断可用信息源 
  确认当前是否可访问小红书、公众号、网页搜索、固定网站或本地缓存。

3. 主动检索 
  使用目标账号 + 活动关键词组合检索。

4. 判断是否为有效活动 
  至少满足以下两项才纳入：
  - 有活动名称
  - 有活动时间
  - 有报名方式
  - 有主办方
  - 有比赛 / 项目 / Demo / 提交作品机制
  - 有奖金、评审、权益或展示环节

5. 去重排序 
  合并重复活动，优先保留官方、完整、未截止的信息。

6. 输出结果 
  按统一字段输出，并说明信息源限制。

## Fallback Strategy

当网络、MCP、API、搜索工具或指定 skill 不可用时，执行兜底策略。

### 兜底原则

- 不直接停止任务。
- 不默认要求用户提供材料。
- 不编造检索结果。
- 不声称已检索失败平台。
- 不重复调用失败工具超过 2 次。
- 必须说明当前受限来源。
- 优先切换到其他可用搜索通道。

### 自动降级顺序

按以下顺序尝试：

1. 微信公众号检索
2. 小红书检索
3. BuilderLab 活动聚合页
4. 公开网页搜索
5. 官方活动页 / 官方报名页
6. 其他可用活动聚合页或搜索入口
7. 本地缓存 / 历史记录 / 已保存结果

### 小红书不可用时

如果小红书 MCP、接口、Cookie、搜索能力不可用：

- 不得声称已检索小红书。
- 改用公开网页搜索。
- 使用关键词：`小红书账号名 + 黑客松 / Hackathon / AI 活动 / 开发者大赛`。

输出说明：

```text
当前环境无法直接访问小红书，已改用公开网页搜索补充检索；本次结果不代表完整覆盖小红书内容。
```

### 微信公众号不可用时

如果微信公众号搜索、文章读取、搜狗微信搜索或授权能力不可用：

- 不得声称已检索公众号。
- 改用公开网页搜索。
- 使用关键词：`公众号名 + 黑客松 / Hackathon / AI 活动 / 开发者大赛`。

输出说明：

```text
当前环境无法直接访问微信公众号，已改用公开网页搜索补充检索；本次结果不代表完整覆盖公众号内容。
```

### skill 不可用时

如果相关 skill 不可用：

- 不重复调用超过 2 次。
- 改用其他搜索 skill、网页搜索或固定网站。
- 保留原始过滤条件。
- 继续完成活动检索与结构化输出。

输出说明：

```text
当前相关 skill 不可用，已切换到其他可用搜索通道。
```

### 网络不可用时

如果网络或所有搜索工具不可用：

- 不输出“最新活动”结论。
- 不编造活动信息。
- 仅可基于本地缓存、历史记录或已有结果整理。
- 必须说明无法保证时效性。

输出说明：

```text
当前网络或搜索工具不可用，无法完成实时检索；以下结果仅基于已有缓存或历史记录，不保证最新。
```

### 空结果处理

如果检索后没有找到有效活动：

- 不直接说“没有活动”。
- 先说明已尝试的信息源。
- 再说明可能原因。
- 最后说明已执行的扩展检索。

输出格式：

```text
基于当前可访问的信息源，暂未找到符合条件的有效活动。

已尝试来源：
- xxx
- xxx

可能原因：
- 活动暂未发布
- 报名已截止
- 当前环境无法访问部分平台
- 关键词范围过窄

已执行的扩展检索：
- 放宽城市限制
- 放宽赛道限制
- 使用主办方 + 黑客松关键词
- 使用公开网页搜索补充
```

## Anti-Hallucination Rules

必须严格防止模型幻觉。

### 信息来源原则

所有活动信息必须来自可验证来源，包括：

- 官方活动页
- 官方报名页
- 主办方公众号
- 主办方小红书
- 活动聚合页
- 公开网页搜索结果
- 本地缓存 / 历史记录

没有来源支撑的信息，不得输出为事实。

### 字段填写原则

- 明确找到的信息：正常填写。
- 没有找到的信息：填写 `未说明`。
- 无法确认的信息：填写 `待确认`。
- 仅推测的信息：不得填写为事实。
- 不同来源冲突的信息：标注冲突，并优先采用官方来源。

禁止为了让表格完整而补全未知信息。

### 禁止推测

禁止根据经验推测：

- 活动是否仍可报名
- 是否有奖金
- 奖金金额
- 是否报销
- 报销上限
- 是否提供住宿
- 活动具体地址
- 报名截止时间
- 主办方身份
- 官方报名链接
- 适合人群标签

如果来源未写明，只能标注 `未说明` 或 `待确认`。

### 安全表述

禁止说：

- “目前没有相关活动”
- “这是最新活动”
- “已覆盖所有信息源”
- “该活动确定有报销”
- “该活动确定可报名”

应该说：

```text
基于当前可访问的信息源，暂未找到符合条件的活动。
```

```text
当前结果不代表完整覆盖所有小红书和公众号内容。
```

## Field Rules

### registration_link

- 有官方报名页时，优先提供官方报名链接。
- 公众号文章内有报名链接时，优先提取原文链接或跳转链接。
- 小红书笔记只有二维码时，填写：`扫码报名，无公开链接`。
- 多活动平台只找到主页面时，填写最接近活动的详情页。
- 无法确认报名链接时，填写：`未找到公开报名链接`。
- 禁止编造报名链接。

### confidence

- `high`：官方来源 + 时间 / 报名方式明确。
- `medium`：非官方来源，但信息较完整。
- `low`：来源存在，但关键信息缺失。

低置信度活动不得作为强推荐，只能放入“待确认活动”。

---

## Deduplication Rules

内部去重字段：

```text
group_key = {organizer}_{start_time}_{city}
```

去重规则：

- 活动名称相似、主办方相同、时间接近，视为同一活动。
- 优先保留官方来源。
- 优先保留信息更完整版本。
- 优先保留有报名链接版本。

---

## Ranking Rules

排序优先级：

1. 报名尚未截止。
2. 时间距离当前更近。
3. 信息来源更官方。
4. 与 AI Agent / 大模型 / 开发者工具更相关。
5. 有奖金、报销、食宿或资源支持。
6. 大厂主办或大厂赞助。
7. 适合用户当前背景和目标。

## User Suitability

用户背景：

- 3 年 Java 后端开发经验。
- 正在求职 AI Agent 相关岗位。
- 正在运营小红书个人自媒体。
- 经常参加黑客松和 AI 活动。
- 正在建设十堰黑客松社区。
- 目标是积累项目经验、人脉、作品集和内容素材。

优先推荐：

- AI Agent / 大模型 / 开发者工具方向。
- 有实际 Demo 产出的活动。
- 有大厂参与或赞助的活动。
- 有线下交流价值的活动。
- 有报销、食宿或资源支持的活动。
- 可用于小红书内容创作的活动。

谨慎推荐：

- 纯商业路演。
- 纯讲座 / 课程 / 分享会。
- 需要成熟创业项目的活动。
- 门槛过高且无组队机会的活动。

## Optional Manual Parsing Mode

本 skill 默认主动搜索信息，不要求用户手动输入。

只有在以下情况才进入手动解析模式：

- 用户主动提供小红书链接、截图或正文。
- 用户主动提供公众号文章链接、截图或正文。
- 用户明确要求从指定材料中提取活动信息。

进入手动解析模式后，仍按统一字段输出活动信息。

## Verification Checklist

输出前必须检查：

- 是否每个活动都有来源？
- 是否没有来源支撑的信息都标为 `未说明` 或 `待确认`？
- 是否没有编造报名链接？
- 是否没有编造奖金、报销、食宿？
- 是否没有编造报名截止时间？
- 是否没有把推测写成事实？
- 是否说明当前信息源访问限制？
- 是否区分官方来源和非官方来源？
- 是否完成去重？
- 是否过滤掉普通课程、直播、讲座？
- 是否没有输出“已覆盖全部信息源”这类过度结论？

## Forbidden Actions

禁止行为：

- 禁止编造活动。
- 禁止编造报名链接。
- 禁止编造奖金、报销、食宿信息。
- 禁止编造报名截止时间。
- 禁止编造活动地点和主办方。
- 禁止根据经验补全未知字段。
- 禁止把推测内容写成事实。
- 禁止把普通文章误判为活动。
- 禁止把课程、直播、分享会误判为黑客松。
- 禁止在没有小红书访问能力时声称“已检索小红书”。
- 禁止在没有微信公众号访问能力时声称“已检索公众号”。
- 禁止在网络或 skill 不可用时假装任务已正常完成。
- 禁止重复调用失败工具超过 2 次。
- 禁止把“让用户提供链接 / 截图 / 正文”作为默认兜底方案。
- 禁止输出已明显过期且不可报名的活动，除非用户要求历史信息。
- 禁止声称“没有活动”，只能说“基于当前可访问来源暂未找到”。

## Quick Reference

### 什么时候用？

用户要找黑客松、AI 活动、开发者大赛、可报名比赛时使用。

### 查哪些源？

小红书、微信公众号、公开网页、BuilderLab、本地缓存或历史结果。

### 按什么筛？

赛道、奖金、时间、地点、报销、主办方、大厂、是否适合用户。

### 工具不可用怎么办？

先说明受限来源，再自动切换其他搜索通道；不要默认让用户输入材料，不编造结果。

### 输出什么？

活动名称、时间、地点、赛道、奖金、报销、报名链接、来源、置信度、适合度、注意事项。

### 不能做什么？

不能编造活动，不能假装检索过不可访问的平台，不能代替用户报名。
