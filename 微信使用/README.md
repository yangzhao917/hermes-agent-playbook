# 微信使用

> 微信消息机制、意图识别与多任务处理

## 消息队列机制

微信消息进来后：

1. **Agent 在 busy** → 新消息排队，等当前 turn 结束后自动处理
2. **Agent 在 idle** → 新消息立即响应

### 连续发消息

直接发，不用等，队列会自动排，依次执行。

### 打断当前任务

- `/stop` + 新任务 → 强制打断 + 执行新任务
- `/reset` → 清空对话历史重新开始

### 消息分发

微信只发简版（一屏：完成事项 + 明日待办 + 待跟进），飞书存全文。

## 意图识别与打断（待实现）

### 当前行为

微信消息进来后，Agent 无法智能区分：
- "补充信息"类消息 → 应该排队
- "纠正/转向"类消息 → 应该打断当前任务

### 理想行为（Feature Request）

| 场景 | 当前 | 期望 |
|------|------|------|
| 任务运行中，用户发补充消息 | 排队 | 排队（保持安静） |
| 任务运行中，用户发送转向/纠正 | 排队（强制等待） | Agent 自然切换到新任务 |
| 用户明确想停止 | 需 `/stop` | `/stop` 仍作为显式覆盖 |

### 临时解决方案

- 想打断 → 直接发 `/stop` + 新任务
- 想加任务 → 直接发，队列会自动处理
