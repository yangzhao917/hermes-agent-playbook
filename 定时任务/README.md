# 定时任务

> cron 配置规范与复盘机制

## Cron 架构规范

### 核心原则

- **禁止使用 script 字段**：避免文件被误删导致 cron 不可用，统一用 `prompt + skills` 模式
- **prompt 内加 anti-hallucination**：约束只排版不编造，数据来自 API 不来自推理

### 永久 cron 清单

> 🔴 以下 3 个为永久 cron job，不得删除，除非用户明确说出「删除日程提醒」。

| 任务名称 | 运行时间 | 目的 | 状态 |
|----------|----------|------|------|
| 每日晨间日程提醒 | 每天 07:00 | 推送今日日历，有凌晨活动特别提醒 | 🟢 永久运行 |
| 每日复盘总结生成 | 每天 22:30 | 生成复盘文档上传飞书云空间，每日复盘文件夹 | 🟢 永久运行 |
| 每周记忆清理 | 每周日 00:00 | 清理过期/失效的记忆条目、矛盾记录、失效工具记录 | 🟢 永久运行 |

## 复盘总结机制

### 存放位置

飞书云空间 `hermesAgent/每日复盘/`（folder token: `<FOLDER_TOKEN>`）

### 命名规则

`YYYY-MM-DD-复盘总结`（文件名=唯一标识，必须严格按文件名搜索确认）

### 更新频率

每天 22:30 自动生成（cron job: 1368ce888da5）

### 文档板块（按优先级）

| 优先级 | 板块 | 标题 | 格式 |
|--------|------|------|------|
| 🟥 必须 | ✅ 完成事项 | `✅ 完成事项` | 表格：时间、事项、备注 |
| 🟥 必须 | ⏰ 明日待办 | `⏰ 明日待办（日期）` | 表格：时间、事项；滚过来的标注来源 |
| 🟥 必须 | 💰 待跟进 | `💰 待跟进` | 列表，带🔴🟡🟢紧迫度标注 |
| 🟧 建议 | 📅 后续安排 | `📅 后续安排` | 表格：日期、事项 |
| 🟧 建议 | 💡 今天学到的 | `💡 今天学到的` | 列表（用「今天学到的」，非「今日新发现」） |
| 🟨 选填 | ⚡ 今日高光/感受 | `⚡ 今日高光/感受` | 1-2句话，有感触才写 |

### 分发方式

微信只发简版（一屏：完成事项 + 明日待办 + 待跟进），飞书存全文。

### 数据来源

真实飞书日历 + 任务列表，不编造。

### 官方日程原则

赛事/活动官方日程不写入复盘文档，直接用 `lark-cli calendar +create` 入日历。

### 更新机制

创建复盘文档前先用 `lark-cli drive search` 在每日复盘文件夹里搜当天日期的文档：
- **找到了** → 更新现有文档内容（不重复创建）
- **没找到** → 创建新文档

```bash
# 搜索当天复盘
lark-cli drive search --folder-token <FOLDER_TOKEN> --keyword "2026-05-18"
```
